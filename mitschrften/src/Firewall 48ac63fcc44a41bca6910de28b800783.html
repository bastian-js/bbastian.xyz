Technologien zur Netzwerksicherheit

  • Paketfilter: Jedes Paket wird mit festeglegten Kriterien (z.B.: IP-Adresse, Pakettyp, Portnummer = enspricht den OSI-Schichten 3 und 4) verglichen.

    Schnell → nur ein Teil der Verbindung wird geprüft (heutigen Sicherheitsanforderungen nicht gerecht)
    Wenn ein Port blockiert ist (
    Leicht umgehbar) → auf anderen Port zurückgreifen
    Umfangreiche Dienste, wie Cloud-Dienste verwenden Ports (80 und 443), die nicht durch den Filter geregelt sind.
  • Stateful Inspection Firewalls: Erweiterung von Paketfiltern → Erkennt, ob eingehende Datenpakete zu der bestehenden Verbindung (Session) gehören oder nicht (DoS-Angriffe).
    OSI-Schichten 3 und 4 kontrolliert und evtl. Nutzdaten (Layer 7) zugegriffen
    • Wie lange ist eine Session gültig: nur für eine bestimmte Dauer
  • Next-Generation Firewalls bzw. Proxy Firewalls: bestimmte Anwendungen können blockiert/überwacht werden (unabhängig der IP-Adresse oder Ports) → Filter bis zur Anwendungsebene möglich
    Erkennen und Abwehren
    aktueller Bedrohungen erst durch diese Technologien möglich
    • Deep-packet inspection: Typ oder Zweck einer Anwendung werden identifiziert (Payload)
    • Intrusion Prevention System: Überwachung des Netzwerkverkehr auf verdächtige Aktivitäten → anhand von typischer Signaturen bekannter Bedrohungen

Schnittstellen

  • LAN: Local Area Network → privates Netzwerk
  • WAN: Wide Area Network → öffentliches Internet
  • DMZ: Demilitarized Zone → spezielle Schnittstelle, die öffentlich zugängliche Server oder Dienste hostet (Dienste sind hier erreichbar)
    z.B.: eigener Webserver, der aus dem Internet erreichbar ist,
    Intranet-Dienste
  • Opt-Schnittstellen: optionale Schnittstellen → getrennter Anschluss für weitere Netzwerke; zusätzliche Funktionen können hinzugefügt werden

TCP: Transmission Control Protocol

Protokoll, das überwacht, auf welche Art und Weise Daten zwischen Netzwerkkomponenten ausgetauscht werden sollen.

Verbindung: Three Way Handshake-Verfahren

Server erzeugt Endpunkt (Socket) mit seiner IP-Nummer und Portnummer → Quell-IP

Client erzeugt eigenen Socket aus Adresse und eigener Portnummer → Ziel-IP, Ziel-Port

  1. Client schickt SYN-Paket (Synchronize Packet) an Port der Servers (mit seiner IP-Adresse und Quellport).
    Das Paket erhält eine zufälliger
    Sequenznummer (zur Identifikation)
  1. Server stimmt Verbindungsaufbau zu und sendet SYN-ACK-Paket (Synchronize/Acknowledge) zurück.
    Hat eine eigene eigene Sequenznummer und Sequenznummer des Client wird um 1 erhöht des Clients.
  1. Client bestätigt Erhalt von SYN-ACK mit eigenem ACK-Paket.

    Sequenznummer des Servers wird um 1 erhöht. → erste Daten werden übertragen

Erstellen von Regeln

Floating Rules: gelten für alle Schnittstellen

Aliases: Objekte können zusammenfassen bzw. erstellen werden, die für eine Regel genutzt werden → z. B. mehrere Ports, bestimmte Adressen, Netzwerk-Bereiche

Aktionen bei Regeln

  • Pass: leitet Verkehr weiter
  • Reject: weist Verbindungsanfrage zurück → mit Fehlermeldung
  • Block: weist Verbindungsanfrage zurück → keine Fehlermeldung (bricht erst nach Timeout ab)
  • Logging: protokollieren → nur bei wichtigen Regeln (zum Testen)
  • Inspect: zeigt Menge des Daten-Traffics einer bestimmte Regel an → Wird eine Regel ignoriert, bleibt dort.

Regeln/Richtlinien - Grundsätze

  • Updates: automatisch (können nicht vergessen werden) oder manuell (Änderungen können vorab angesehen weren) → Probleme/Fehler durch neue Updates
  • Absichern des Betriebssystem: Bei Firewall-System als Komplettpaket inkl. Betriebssystem ist alles voreingestellt → Möglichst wenig weitere Software installiert
  • Standard-Regeln: sollten erstellt werden → blockiert standardmäßig alles, was nicht erlaubt ist
  • Tests und Sicherheitsüberwachungen: regelmäßiger Test der Einstellungen (vorallem bei Migrationen oder Umstellungen) → nicht von der Verwaltungsperson auszuführen
  • Zugriff auf die Firewall: Einstellungen nur von Administratoren, starke Passwörter → keine Default- und Test-Accounts
  • Sauberes Regelwerk: wenig Regeln, bieten wenig Schlupflöcher → keine Duplikate und obsolete (überflüssige) Regeln; Überschattung überprüfen
  • Logfiles: Regeln, die Log-Einträge machen oder Dienst wird „live” überwacht → wichtige Ereignisse
  • Dokumentation: bei mehreren Personen

Principle of least privilege

Ebenen: Rechte der Benutzer, Zugriff von verschiedenen Geräten untereinander, Zugriff auf Daten von verschiedenen Software-Produkten.

Wartung: Zugriff auf die Firewall aus dem WAN akzeptieren → Interface: WAN | Source: Any | Destination: any

Beispiel-Regel

  • Source: Any
  • Destination: This Firewall (vordefiniertes Objekt bei Opnsense)
  • Protokoll: TCP
  • Port: 443 (HTTPS) → Firewall muss über Port 443 (HTTPS) und nicht Port 80 (HTTP) erreichbar sein

Dadurch ist nur HTTPS möglich → alle anderen Versuche werden blockiert
